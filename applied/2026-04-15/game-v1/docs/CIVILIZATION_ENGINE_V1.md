# Civilization Engine v1

## Purpose
The Civilization Engine links societies, technologies, knowledge transmission, governance, collapse pressure, and reactivation into one governed simulation layer.

## Core law
A civilization is not a static label. It is a dynamic system of actions, institutions, technologies, memory, and feedback loops.

## Core state vector
C = (G, K, T, M, I, E, P)

Where:
- G = governance quality
- K = knowledge continuity
- T = technical capacity
- M = memory integrity
- I = inclusion and justice balance
- E = ecological/resource stability
- P = collapse pressure

## Main loop
Society profile -> Technologies -> Actions -> Consequences -> System state update -> Review gate -> Public summary or simulation branch

## Output rails
- real-history interpretation support
- hypothesis under review support
- simulation and playable branch generation

## Collapse and reactivation law
A civilization may gain local power while increasing future collapse pressure.
A civilization may lose short-term power while increasing long-term survivance.

## Example strategic questions
- Which technologies improve memory without increasing domination?
- Which institutional forms create strong capacity but hidden fragility?
- Which action patterns increase collapse pressure across generations?
- Which combinations support reactivation after loss?

## Final law
The Civilization Engine exists to make long-range consequences legible, not to flatten history into a single moral score.
