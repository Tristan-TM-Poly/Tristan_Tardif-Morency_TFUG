# Rosetta-Tristan Stone Ledger v1

## Purpose
This ledger governs alignment across inscriptions, symbols, transliterations, translations, procedures, and functional hypotheses.

## Core law
A symbol is not automatically a word, and a word is not automatically a procedure.

## Rails
- real-history alignment
- hypothesis alignment
- simulation/lore alignment

## Required fields per item
- id
- source_object
- civilization_or_context
- source_form
- transliteration
- translation_candidates
- symbol_families
- procedural_candidates
- invariant_terms
- ambiguity_terms
- alignment_score
- rail_status
- review_gate_status
- citations_required
- anti_inflation_notes

## Status values
- established-alignment
- strong-alignment
- working-alignment
- contested-alignment
- speculative-alignment
- lore-alignment-only

## Example entries

## Output use
This ledger feeds:
- Rosetta alignment surfaces on the site

- TRISTAN2 anti-inflation review for symbol/function claims

## Final law
The Rosetta-Tristan Stone Ledger is where translation becomes disciplined comparison instead of symbolic wishful thinking.
