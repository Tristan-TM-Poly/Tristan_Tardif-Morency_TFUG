# Civilization Engine v2

## Purpose
Extend the first Civilization Engine into a richer governed simulator with nonlinear interactions, external shocks, adaptive stabilization, and gameplay-ready branching.

## Core law
A civilization is a nonlinear system. Gains in one dimension can generate hidden fragilities in another.

## State vector
C = (G, K, T, M, I, E, P)

Where:
- G = governance quality
- K = knowledge continuity
- T = technical capacity
- M = memory integrity
- I = inclusion and justice balance
- E = ecological/resource stability
- P = collapse pressure

## New v2 features
- nonlinear interaction terms
- external shock events
- recovery and repair dynamics
- adaptive recommendations
- scenario branching for gameplay

## Example nonlinear principles
- rapid technical increase without ecological stability can sharply increase collapse pressure
- high memory plus high governance can reduce collapse pressure faster than either alone
- high governance with low inclusion may produce deceptive short-term order and long-term fragility
- high knowledge continuity supports faster recovery after shocks

## Shock families
- war shock
- climate shock
- resource shock
- institutional shock
- memory loss shock
- knowledge breakthrough shock

## Output layers
- evaluation layer
- scenario layer
- gameplay recommendation layer
- review gate support layer

## Final law
Civilization Engine v2 is not a truth machine. It is a governed simulator for comparing trajectories, tensions, collapse risks, and reactivation paths.
