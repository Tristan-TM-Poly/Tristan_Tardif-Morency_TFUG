# FRONT_AUTO_IMPROVEMENT_GENERATION_POLICY — Applied Snapshot (2026-04-14)

## Purpose
Define how automatic front improvements and automatic module generation should operate inside the TFUGA public application layer.

## Core rule
Front auto-improvement must be orchestrated as governed loops rather than isolated file accumulation.

## Required loop
absorb -> canonize -> generate -> propagate -> test -> cleanup -> reindex

## Required governors
- scorecards
- review gates
- maturity distinctions
- rollback path
- branch-aware reports
- release lineage

## Allowed outputs
- improved public routes
- improved generated modules
- improved deployment readiness notes
- improved AI-BOT and Yggdrasil surfaces
- improved membership and dashboard surfaces

## Law
Front auto-improvement is valid only if it increases closure, navigability, and publication readiness faster than it increases front complexity bulk.
