# AUTO_GENERATOR_WEB_INTEGRATION_POLICY — Applied Snapshot (2026-04-14)

## Purpose
Define how auto-generators should create, code, and integrate bounded modules into the public interactive website while remaining coupled to TFUGA x AI-7 x TRISTAN2.

## Core rule
An auto-generator is valid only if it can generate a bounded public module together with:
- one TFUGA object or mechanic
- one AI-7 interaction or adaptation loop
- one TRISTAN2 review or gate layer
- one public conversion path

## Allowed generator outputs
- demo cards
- bounded visualizers
- page sections
- navigation fragments
- collaboration CTA modules
- public artifact viewers

## Required safeguards
- generator outputs must stay interpretable
- generator outputs must point to governed artifacts or packet families
- generator outputs must not bypass review, score, or progression logic
- generator outputs should remain small, composable, and auditable

## Law
Auto-generators succeed only when they increase production speed, public clarity, and canon coupling at the same time.
