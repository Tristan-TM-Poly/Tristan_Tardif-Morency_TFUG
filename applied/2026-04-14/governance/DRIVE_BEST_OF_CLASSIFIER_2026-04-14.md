# DRIVE_BEST_OF_CLASSIFIER — Applied Snapshot (2026-04-14)

## Classes
### Class A — GitHub-now
Artifacts that are trunk-facing, reusable, benchmarkable, reviewable, code-facing, or publication-prep-critical.

### Class B — GitHub-soon
Artifacts that are crystallizable and likely to become active source-of-truth once one more closure step is attached.

### Class C — Drive-primary
Artifacts that are useful, rich, or exploratory, but not yet current executable memory.

### Class D — Drive-archive
Artifacts that are historically useful or broad archive mass with low current closure value.

## Best-of law
The best Drive objects should concentrate into GitHub only when doing so increases closure, reproducibility, and current-use clarity more than it increases bulk.

## Current default reading
- sovereign trunk and runtime-facing governance: Class A
- Fractal-Loop pilot artifacts: Class A
- bounded LDK analytic companion artifacts: Class A/B
- Raman frontier artifacts: Class B moving toward A
- broad archive-heavy bundles: Class D
