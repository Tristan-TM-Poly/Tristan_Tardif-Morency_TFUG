# AUTO_GOVERNED_EXPONENTIAL_IMPROVEMENT_POLICY — Applied Snapshot (2026-04-14)

## Purpose
Define how broad auto-improvement and auto-generation waves should run across the TFUGA sovereign stack without collapsing governance.

## Core rule
Exponential generation is allowed only as bounded branching under explicit review, score, packet, and rollback discipline.

## Allowed expansion axes
- packets and packet families
- public pages and bounded demo modules
- educational game routes and NPC guidance layers
- Yggdrasil / hypergraph growth surfaces
- AI-7 and TRISTAN2 runtime support artifacts
- research, thesis, and publication support artifacts

## Required governors
- scorecards
- review gates
- contradiction routes
- rollback and quarantine pointers
- reduced baselines
- public maturity distinctions

## Exponential law
Branching must increase closure density faster than symbolic bulk.
If bulk rises faster than closure, the branch must hold, compress, or rollback.
