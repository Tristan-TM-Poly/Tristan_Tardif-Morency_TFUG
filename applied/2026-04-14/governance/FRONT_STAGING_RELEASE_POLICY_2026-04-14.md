# FRONT_STAGING_RELEASE_POLICY — Applied Snapshot (2026-04-14)

## Purpose
Define how the interactive site should move from release candidate status toward staged public deployment.

## Staging layers
- local webapp scaffold
- internal release candidate front
- bounded staging deployment
- public release candidate
- broader public deployment

## Required checks before advancing
- preferred front entry identified
- unified navigation verified
- strongest public routes verified
- membership and dashboard routes reviewed
- maturity distinctions preserved
- rollback path documented

## Law
A front staging release is valid only if each promotion step increases real usability and deployment readiness without masking unresolved integrity gaps.
