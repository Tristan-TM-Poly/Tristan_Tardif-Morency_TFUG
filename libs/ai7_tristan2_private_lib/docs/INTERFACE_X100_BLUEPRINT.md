# Interface X100 Blueprint

## Brutal diagnosis
The current private library is a strong sovereign base, but not yet an x10/x100 orchestration interface.
The next multiplier is not more documents. It is a higher-level orchestration membrane.

## X100 objective
Create an interface layer that lets Tristan, AI-7, TRISTAN2, and Drive-governed workflows operate through one command grammar.

## Core multipliers
1. unified command interface
2. explicit task queue and task graph
3. batch orchestration for repeated operations
4. adapters for Drive, GitHub, scoreboards, and packets
5. stateful session memory for current campaign context
6. fast promotion / quarantine decisions
7. pilot closure routing
8. contradiction handling and downgrade routing

- `cli.py`
- `command_router.py`
- `task_queue.py`
- `planner.py`

- `adapters/drive_adapter.py`

## Unified command examples
- `ai7 push pilot fractal-loop`
- `tristan2 score artifact TFUGA-BOOT-001`
- `drive attach doc <doc_id> to artifact <artifact_id>`
- `governance promote artifact <artifact_id>`
- `campaign create bounded-raman-review`
- `contradiction open artifact <artifact_id>`

## X100 speed principle
The interface must reduce repeated orchestration overhead by turning many small manual decisions into reusable commands and batch operations.

## Quality principle
The interface must not only be faster. It must also increase:
- consistency
- auditability
- reality tagging
- packet completeness
- k-synergy extraction
- canon-facing compression

## Immediate implementation direction
Build the orchestration interface first inside the private library, then let future repos import it as the sovereign SDK.
