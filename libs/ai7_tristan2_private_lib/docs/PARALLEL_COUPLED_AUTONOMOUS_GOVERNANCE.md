# Parallel Coupled Autonomous Governance

## Permanent doctrine
Always prefer:
- parallel execution when dependency-safe
- coupled execution when branch gains reinforce each other
- autonomous governance checks before promotion or widening
- quarantine and downgrade paths when contradictions or unresolved dependencies appear

## Operational chain
1. define campaign
2. generate tasks
3. form coupled parallel groups
4. run autonomous governance evaluation
5. execute approved groups
6. review outputs
7. update ledger and score rows
8. promote, downgrade, or quarantine

## Safety rule
Parallelism is native, but never blind.
If unresolved dependencies exist, autonomous governance may switch from coupled parallel to safe sequential.
