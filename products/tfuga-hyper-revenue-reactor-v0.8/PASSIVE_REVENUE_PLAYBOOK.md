# Passive Revenue Playbook

Status: T3 semi-passive revenue architecture for human review.

Purpose: turn TFUGA / AI-7 / HGFM artifacts into low-maintenance digital revenue candidates without claiming guaranteed passive income, autonomous revenue, or market validation before receipts exist.

## Core law

Passive revenue is not automatic money. It is a system that can keep selling or receiving support after the initial setup only if the product, proof, distribution, trust, delivery, support, refund/privacy path, and maintenance loop exist.

## Revenue equation

SemiPassiveRevenue = ProductValue x Trust x Distribution x Checkout x Delivery x Support x Maintenance x Evidence / (Hype x Confusion x Risk x RefundFriction x MaintenanceDebt).

## Minimum viable revenue path

1. Select one offer.
2. Write a clear one-page product promise.
3. Route all claims through ClaimRouter.
4. Add support/privacy/refund notes.
5. Prepare delivery artifact.
6. Ask 5-10 relevant reviewers privately.
7. Record feedback packets.
8. If someone sponsors or pays, record anonymized receipt/sponsor log.
9. Deliver artifact and record delivery evidence.
10. Promote only to R4-candidate after review.

## Maintenance rule

A revenue asset is only semi-passive if maintenance is bounded.

For each asset define:

```text
support load per buyer
update frequency
refund/cancel path
known limitations
versioning policy
```

## Hard law

No receipt, no revenue proof. No support path, no public selling. No maintenance plan, no passive claim.
