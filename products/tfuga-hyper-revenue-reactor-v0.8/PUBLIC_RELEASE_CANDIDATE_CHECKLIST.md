# Public Release Candidate Checklist

Status: `T3-release-candidate-gate`.

This checklist defines the minimum conditions before moving the HyperRevenue Reactor / OmniSpectral EvidenceGate Starter Kit from draft PR to public release candidate.

## Required before ready-for-review

- [ ] CI governance checks are green.
- [ ] Support contact exists.
- [ ] Refund note reviewed.
- [ ] Privacy note reviewed.
- [ ] Delivery manifest matches the paid offer.
- [ ] Checkout copy says exactly what the buyer receives.
- [ ] No guaranteed passive income language.
- [ ] No claim that Stripe, Drive, GitHub, taxes, privacy, or legal compliance are already solved unless evidence exists.
- [ ] Human review approval exists.

## Required before T4 claim

- [ ] Payment processor link or sponsor mechanism exists.
- [ ] First real customer receipt or sponsor log exists.
- [ ] Receipt is anonymized or hashed before being committed.
- [ ] Delivery was completed.
- [ ] Support/refund status is recorded.
- [ ] Customer feedback or sponsor note is summarized without private data.

## Required before T5 claim

- [ ] Three or more independent customers or sponsors.
- [ ] Repeat purchase, retention, or second-cycle interest.
- [ ] Basic accounting export exists.
- [ ] Maintenance/update policy exists.
- [ ] Support load is measured.

## Required before T6 claim

- [ ] 30-90 days stable low-maintenance revenue.
- [ ] Refund rate below target.
- [ ] Support load below target.
- [ ] No unresolved compliance or safety blocker.
- [ ] Versioned delivery pipeline.

## Hard law

A public release candidate can be prepared by documentation and CI. A revenue claim requires real receipt evidence.
