# TFUGA x10 Amplification Protocol

Status: S3/T3/K3 governed amplification architecture.

Purpose: multiply the useful capacity of TFUGA / AI-7 / HGFM by 10 without multiplying hype, unsafe autonomy, unsupported claims, duplication, risk, or external actions.

## Core law

Everything x10 means evidence passports x10, tests x10, logs x10, feedback x10, reuse x10, and clarity x10. It does not mean claims x10.

## x10 equation

X10 = Gate(10 x Value) where Gate = HumanReviewGate + EvidenceGate + ClaimRouter + FeedbackLoop + ContinuousGovernance + Rollback.

A branch can be amplified only if:

ValueGain_x10 > Complexity_x10 + Risk_x10 + Duplication_x10 + Maintenance_x10.

## Ten amplification axes

1. Agents x10: more bounded agent specs, not uncontrolled agents.
2. Theories x10: more TheoryPackets with assumptions, equations, tests, falsifiers.
3. Systems x10: more SystemBridgePackets with invariants, failure modes, prototype paths.
4. Applications x10: more ApplicationPackets routed by domain, user, impact, risk, evidence.
5. Evidence x10: more evidence packets, missing-evidence lists, hashes, controls, receipts.
6. Tests x10: more CI, dry-runs, schema checks, sample packets, regression tests.
7. Feedback x10: more human/AI comments converted into structured feedback packets.
8. Visibility x10: more clear public-safe artifacts, not exaggerated claims.
9. Reuse x10: more reusable templates, schemas, dashboards, and protocols.
10. Crystallization x10: more compression into stable kernels, less duplication.

## Multiplication rule

For each artifact A, create at most ten derivatives:

- 3 proof derivatives;
- 2 product derivatives;
- 2 education/visibility derivatives;
- 1 ethics/respect derivative;
- 1 dashboard derivative;
- 1 rollback/archive derivative.

If any derivative lacks a gate, evidence path, or blocked-claims section, it is not allowed.

## x10 packet

Every x10 action must define:

```text
amplification_id
source_artifact
axis
factor
allowed_derivatives
required_gate
required_evidence
blocked_claims
risk_control
rollback_path
next_least_action
```

## Blocked x10

Do not multiply:

- unsupported scientific claims;
- revenue claims without receipts;
- cultural/community knowledge without review/permission;
- public launches without human approval;
- agents without specs/tests/rollback;
- documentation that duplicates existing canon without compression;
- workflows with write permissions or hidden external actions.

## x10 least-action route

1. Route claim.
2. Generate packet.
3. Add gate.
4. Add tests.
5. Add feedback intake.
6. Add dashboard card.
7. Run continuous governance.
8. Compress into canon.
9. Archive or block weak derivatives.
10. Promote only with evidence.

## Hard law

Multiply capacity, not noise. Multiply evidence, not claims. Multiply routes to reality, not declarations of victory.
