# TFUGA Governance Canon

Status: S3/T3/K3 compact canon for human review.

Purpose: compress the many governance layers of PR #30 into one small, reusable canon. This document is the map, not the territory. It does not merge, deploy, publish, spend, create payment links, validate science, validate revenue, or authorize cultural/community knowledge use.

## Canon equation

GovernanceCanon = ClaimRouter + EvidenceGate + HumanReviewGate + ContinuousGovernance + GovernanceDashboard + FeedbackLoop + TransactionGate + KnowledgeRespectGate + Rollback.

## Core law

Create many. Route everything. Prove gradually. Promote only what survives the correct gate.

## Nine canonical gates

1. ClaimRouterGate
   - Classifies each claim as science, software, revenue, cultural, product, deployment, governance, or unknown.

2. EvidenceGate
   - Blocks scientific/software/status promotion without evidence, tests, data, uncertainty, or reproducibility.

3. RevenueEvidenceGate
   - Blocks T4/T5 revenue claims without real receipt, sponsor log, delivery evidence, support path, privacy/refund review.

4. KnowledgeRespectGate
   - Blocks cultural/community/place-based knowledge extraction, ownership, unauthorized translation, and commercial use without review/permission.

5. HumanReviewGate
   - Blocks external actions: merge, deploy, publish, payment link, outreach, purchase, subscription, or public launch.

6. ContinuousGovernanceGate
   - Runs all available local/read-only governance checks and emits status artifacts.

7. GovernanceDashboardGate
   - Turns gate results into readability: status, readiness score, blockers, next least-action route.

8. FeedbackLoopGate
   - Converts human/AI comments into improvement packets, not proof.

9. TransactionGate
   - Routes revenues and purchases through budget, risk, ledger, evidence, approval, and rollback/cancel paths.

## Status ceilings

- S3 maximum for documents, code architecture, local tools, dashboards, dry-runs, and review-stage software without real external validation.
- T3 maximum for product/revenue plans without a real receipt or sponsor log.
- K3 maximum for public educational cultural/community references without appropriate review or permission.
- R3 maximum for revenue candidates without receipt/sponsor evidence.
- P1/P2 maximum for purchase candidates without explicit approval and budget review.

## Promotion laws

Science promotion requires real data, metadata, controls, calibration, uncertainty, hashes, and reproduction path.

Revenue promotion requires real buyer/sponsor evidence, delivery proof, support path, and privacy/refund review.

Cultural/community promotion requires attribution, review path, benefit-back, permission where needed, and no restricted content.

Deployment promotion requires explicit human approval, rollback, and audit trail.

## Blocked universal claims

- guaranteed passive income;
- autonomous revenue;
- validated physics without data;
- automatic deployment/publication/payment;
- ownership or absorption of community/cultural knowledge;
- AI replacement of human/community/scientific review;
- x10 amplification as truth;
- dashboard status as external proof.

## Least-action route

1. Write the claim.
2. Route the claim.
3. Assign the gate.
4. List missing evidence.
5. Patch or block.
6. Run continuous governance.
7. Read dashboard.
8. Ask for human review.
9. Promote only with evidence.
10. Archive weak branches.

## Hard law

The canon stays small. The ecosystem may grow, but truth only moves through gates.
