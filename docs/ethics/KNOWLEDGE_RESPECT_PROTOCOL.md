# Knowledge Respect Protocol

Status: S3 ethical documentation architecture.

Purpose: provide a safe review gate for any project that references community knowledge, cultural knowledge, language material, ecological observations, oral history, or place-based practices.

Core rule: knowledge is not raw data. Public summaries can be discussed with attribution. Restricted, private, ceremonial, community-governed, or sensitive material requires permission and review before use.

Gate equation:

UseAllowed = PublicOrPermitted AND ContextPreserved AND BenefitBackDefined AND AttributionPresent AND PurposeLimited AND ReviewPathDefined.

Status ladder:

K0 raw idea.
K1 public source identified.
K2 general educational summary.
K3 public teaching artifact with attribution and limits.
K4 reviewed by appropriate knowledge holder or community reviewer.
K5 formal partnership and benefit sharing.
K6 stable co-governed protocol with audit and withdrawal rights.

Automatic blockers:

- private or restricted material;
- community-specific material without permission;
- sensitive location information;
- health or plant-use claims presented as advice;
- commercial use without agreement;
- claims that TFUGA or AI owns, absorbs, restores, or canonizes community knowledge.

HGFM structure:

Nodes: source, claim, permission status, intended use, benefit plan, risk class, review status, evidence status.

Hyperedges: claim + source + attribution + limitation + status; proposed use + permission status + benefit plan + review path; educational module + public principle + no-sensitive-content + citation.

Hard law: respect first, permission second, documentation third, application last.
