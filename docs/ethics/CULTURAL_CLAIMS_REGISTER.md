# Cultural / Community Knowledge Claims Register

Status: S3 ethical documentation architecture.

Purpose: keep every claim about community, cultural, language, ecological, oral, or place-based knowledge under a respect-first gate.

## Allowed claims

| Claim | Status | Evidence required |
|---|---|---|
| This project defines a respect protocol. | allowed | protocol document |
| Public sources may be summarized with attribution and limits. | allowed | public source + citation + limitations |
| Sensitive or private material requires permission and review. | allowed | respect protocol |
| Commercial use requires an explicit agreement. | allowed | agreement record before use |
| Community review can promote an artifact from K3 to K4. | allowed | review note or approved process |

## Careful claims

| Claim | Status | Evidence required |
|---|---|---|
| A public principle can inspire a design analogy. | careful | source + clear statement that it is an analogy |
| A module may support future collaboration. | careful | contact/review path + benefit-back plan |
| A translation can be useful. | careful | translator/reviewer and allowed audience |
| A knowledge graph can document relationships. | careful | no restricted content + attribution |

## Blocked claims

| Claim | Status | Reason |
|---|---|---|
| TFUGA owns community knowledge. | blocked | ownership and governance remain with the relevant people/community/source holders |
| AI replaces Elders, Knowledge Keepers, or community review. | blocked | AI cannot replace human authority and consent |
| The system translates everything. | blocked | translation requires permission, context, and review |
| Sensitive or restricted knowledge may be used as technology. | blocked | prohibited without explicit authority and review |
| Community knowledge may be commercialized without agreement. | blocked | violates benefit-back and consent gates |
| Documentation alone promotes K4/K5/K6. | blocked | review, partnership, and governance evidence required |

## Promotion law

K0-K3 can be built from public, general, non-sensitive educational sources with attribution.

K4 requires review by an appropriate knowledge holder, language holder, or community review path.

K5 requires formal partnership and benefit sharing.

K6 requires stable co-governed protocol, audit, and withdrawal or correction rights.

## Hard law

Respect first. Permission second. Documentation third. Application last.
