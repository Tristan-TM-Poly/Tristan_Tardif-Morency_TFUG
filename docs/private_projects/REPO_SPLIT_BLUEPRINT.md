# Repo Split Blueprint

## Why split
A single private repository can host the control nucleus, but long-term power comes from bounded sovereign private repositories with explicit roles.

## Proposed split
### 1. tfuga-canon-private
Purpose: stable canon, notes fondatrices, validated spines, theorem kernels, proof-facing artifacts.

### 2. tristan2-runtime-private
Purpose: runtime bus, packet schemas, governance organs, review engines, version gates.

### 3. ai7-execution-private
Purpose: AI-7 execution loops, campaign runners, optimization and self-healing logic.

### 4. fractal-loop-pilots-private
Purpose: Fractal-Loop closed pilots, reduced models, score rows, contradiction packets.

### 5. ldk-paf-projector-private
Purpose: bounded analytic examples for LDK, PAF, Projector Fractal, Exp(*).

### 6. research-thesis-private
Purpose: thesis ladders, article packets, note conversion, publication outputs.

### 7. energy-core-private
Purpose: bounded physical incarnation only after closure gates are satisfied.

## Shared policy across all repos
- every repo must define its source of truth
- every repo must maintain a pilot closure queue
- every repo must expose score rows for strong modules
- every repo must maintain theory-to-runtime conservation tracking
- every repo must support downgrade as well as promotion

## Minimal file skeleton for each future private repo
- README.md
- SYSTEM_ROLE.md
- PILOT_CLOSURE_QUEUE.md
- SCORE_RULES.md
- RISK_REGISTER.md
- REALITY_MAP.md
- THEORY_TO_RUNTIME.md
- K_SYNERGY_MATRIX.md
- PROMOTION_LEDGER.md

## Current bridge strategy
Until direct private repository creation is available through the connector, continue using this private repo as a sovereign bootstrap nucleus and grow subdirectories that can later be split into dedicated private repos.
